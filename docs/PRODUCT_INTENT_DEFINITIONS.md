# Role-Based Product Intent Definitions

## General User

### Primary Job
**Find and connect with trusted construction partners and dealers**

The app exists to help users discover, evaluate, and contact the right professionals for their construction needs.

### Secondary Jobs
- Get construction estimates and budget calculations
- Access educational resources and checklists
- Track enquiries and responses
- Save favorite partners/dealers for future reference

### Daily Success Moment
**"I found a verified partner who responded to my enquiry within 2 hours"**

### Failure / Friction Moments
- Too many search results without clear differentiation
- No response to enquiries (feels ignored)
- Unclear partner/dealer verification status
- Confusing navigation between search, profile, and enquiry

### Primary CTA per Key Screen
- **Dashboard**: "Search Partners" or "Search Dealers" (one primary, one secondary)
- **Search Results**: "View Profile" (most prominent)
- **Profile**: "Send Enquiry" (primary action)
- **Enquiries**: "View Response" (if responded) or "Follow Up" (if pending)

### Dashboard Focus
- **Primary Card**: Search prompt (Partners or Dealers based on user preference)
- **Supporting Cards**: Recent searches, Pending enquiries count, Favorite count
- **Hidden by Default**: Detailed analytics, historical data, advanced filters

### Default Screen Priority
1. Search (if no recent activity)
2. Enquiries (if pending responses)
3. Favorites (if user has saved items)

## Partner

### Primary Job
**Showcase work, respond to enquiries, and grow reputation**

Partners need to demonstrate expertise and convert enquiries into projects.

### Secondary Jobs
- Manage portfolio and media
- Track enquiry response rates
- View feedback and ratings
- Access business tools (checklists, calculators)

### Daily Success Moment
**"I received 3 new enquiries and responded to all within 1 hour"**

### Failure / Friction Moments
- Enquiries get lost in a cluttered inbox
- No clear feedback on response quality
- Difficulty showcasing best work
- Unclear how to improve visibility

### Primary CTA per Key Screen
- **Dashboard**: "View New Enquiries" (if any) or "Add Work" (if none)
- **Works**: "Add New Work" (primary)
- **Enquiries**: "Respond" (for new enquiries)
- **Profile**: "Edit Profile" (secondary, not primary)

### Dashboard Focus
- **Primary Card**: New enquiries count with quick response action
- **Supporting Cards**: Recent works, Feedback summary, Response rate
- **Hidden by Default**: Analytics details, historical reports, advanced settings

### Default Screen Priority
1. Enquiries (always first - this is the revenue driver)
2. Works (portfolio management)
3. Profile (maintenance, not primary)

---

## Dealer

### Primary Job
**Manage product catalogue, respond to enquiries, and drive sales**

Dealers need to showcase products and convert enquiries into sales.

### Secondary Jobs
- Manage product inventory
- Track enquiry-to-sale conversion
- View customer feedback
- Access product tools

### Daily Success Moment
**"I responded to 5 enquiries and closed 2 sales today"**

### Failure / Friction Moments
- Product catalogue is hard to manage
- Enquiries get missed
- No clear product performance metrics
- Unclear which products drive enquiries

### Primary CTA per Key Screen
- **Dashboard**: "View New Enquiries" (if any) or "Add Product" (if none)
- **Products**: "Add Product" (primary)
- **Enquiries**: "Respond" (for new enquiries)
- **Profile**: Secondary action only

### Dashboard Focus
- **Primary Card**: New enquiries with response action
- **Supporting Cards**: Product count, Enquiry conversion rate, Recent feedback
- **Hidden by Default**: Detailed analytics, historical data, advanced reports

### Default Screen Priority
1. Enquiries (revenue driver)
2. Products (inventory management)
3. Profile (maintenance)

---

## Admin

### Primary Job
**Ensure platform quality, moderate content, and support growth**

Admins maintain trust and quality across the platform.

### Secondary Jobs
- Approve/reject partner/dealer applications
- Moderate feedback and content
- Monitor platform health
- Manage system configuration

### Daily Success Moment
**"I approved 5 new partners, resolved 3 reports, and the platform is healthy"**

### Failure / Friction Moments
- Too many pending approvals (feels overwhelming)
- Unclear what needs attention first
- No clear priority system
- Difficult to find specific users/content

### Primary CTA per Key Screen
- **Dashboard**: "Review Pending Applications" (if any) or "View Reports" (if any)
- **Users**: "Approve" or "Reject" (for pending)
- **Content**: "Moderate" (for flagged content)
- **Reports**: "Resolve" (for open reports)

### Dashboard Focus
- **Primary Card**: Pending actions requiring attention (approvals, reports)
- **Supporting Cards**: Platform health KPIs, Recent activity summary
- **Hidden by Default**: Detailed analytics (show on demand), Historical data

### Default Screen Priority
1. Pending Actions (what needs attention NOW)
2. Platform Health (monitoring)
3. Content Management (maintenance)

## Intent Translation Rules

### Screen Design Principles
1. **One Primary Action**: Each screen has ONE visually dominant action
2. **Clear Hierarchy**: Primary > Secondary > Tertiary (hidden)
3. **Context-Aware**: Show what matters NOW, hide what doesn't
4. **Progressive Disclosure**: Advanced features behind navigation

### What NOT to Show by Default
- Historical data older than 30 days
- Advanced analytics (show on demand)
- Configuration options (settings menu)
- Experimental features
- Redundant information

### Success Metrics per Role
- **General User**: Time to find partner, enquiry response rate
- **Partner**: Enquiry response time, conversion rate
- **Dealer**: Enquiry response time, product views
- **Admin**: Time to resolve pending actions, platform health
